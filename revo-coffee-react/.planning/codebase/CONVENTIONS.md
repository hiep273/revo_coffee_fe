# Code Conventions

- **Component Style**: Functional components with React Hooks.
- **File Naming**: PascalCase for React components (e.g., `Header.jsx`, `ProductDetail.jsx`), camelCase for utilities/store (e.g., `useStore.js`, `i18n.js`).
- **Styling**: Tailwind CSS utility classes used directly in JSX components.
- **Linting**: ESLint is configured with React hooks and React refresh plugins.
- **Module System**: ES Modules (`"type": "module"` in `package.json`).
